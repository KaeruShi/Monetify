Contributing ​

Contributions are always welcome! Whether you are fixing a typo, improving documentation, refactoring code, or adding a new feature, every contribution helps.

How to contribute ​

  1. Fork the repository
  2. Create a feature branch
  3. Make your changes
  4. Open a pull request

Guidelines ​

  • Check for existing issues before opening a new one.
  • Include clear descriptions and steps to reproduce bugs.
  • Provide logcat output for runtime crashes when possible.

Roadmap ​

Planned features

  • Shizuku support
  • More app support
  • More icon packs
  • More customization options
  • Custom icons
